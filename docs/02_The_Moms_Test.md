# 02 · The Mom's Test：如何做不被欺骗的用户访谈？

> **目标：** 在写代码之前，用 5 次对话排除「他们会买的」式幻觉。  
> 参考 Rob Fitzpatrick《The Mom's Test》，结合浪前访谈现场提炼。

---

## 一、三条铁律

1. **谈他们的过去，不谈你的未来** — 「上周你怎么解决这个问题的？」
2. **谈行为，不谈观点** — 「你为此付过钱吗？」优于「你觉得有用吗？」
3. **别问妈妈** — 亲近的人一定会骗你（出于爱）

---

## 二、坏问题 vs 好问题

| ❌ 坏问题（会得到谎言） | ✅ 好问题（可核对） |
|------------------------|---------------------|
| 你会用这个产品吗？ | 上次遇到这个问题是什么时候？ |
| 你觉得 idea 怎么样？ | 你现在怎么解决？花了多少钱 / 多少时间？ |
| 要是我们做出来了你会买吗？ | 如果没有更好的方案，你会继续忍受吗？ |
| 你需要哪些功能？ | 哪一步最让你烦躁？ |

---

## 三、5 人规则

MVP 前至少聊 **5 个目标用户**，其中 **≥2 人不是你的朋友**。

| 人数 | 你应该看到什么 |
|------|----------------|
| 1–2 | 可能只是礼貌 |
| 3–4 | 开始重复类似的痛点 |
| 5+ | 若仍无重复，**换人群或换问题** |

每次访谈 **≤30 分钟**，录音前征得同意，24 小时内整理笔记。

---

## 四、访谈脚本（可直接用）

```text
开场（1 min）
「我在研究 [场景]，不推销任何东西，想听听你的真实经历。」

过去行为（15 min）
1. 最近一次 [痛点场景] 是什么时候？
2. 当时你怎么处理的？
3. 为此花过钱吗？用的什么工具？
4. 最烦的一步是什么？

深挖（10 min）
5. 如果一直不解决，会怎样？
6. 你试过别的方案吗？为什么放弃了？

收尾（2 min）
「还有谁也有类似困扰？能介绍我认识吗？」
```

完整模板见 [toolkit/interview-template.md](../toolkit/interview-template.md)。

---

## 五、笔记怎么写才有用

每次访谈填四格：

| 字段 | 写什么 |
|------|--------|
| **事实** | 他们 **做过** 什么（可核对） |
| **代价** | 时间 / 金钱 / 情绪成本 |
| **引用** | 原话一句（脱敏） |
| **我的假设** | 被证实 / 被推翻 |

若「事实」栏为空，这次访谈 **不算数**。

---

## 六、常见自欺信号

- 访谈完更兴奋，但没人掏过钱
- 用户说「挺好」但从没问「什么时候能用」
- 你讲了 20 分钟产品，对方只说了「厉害」
- 5 个人里有 5 种完全不同的痛点 — 说明你还不够聚焦
